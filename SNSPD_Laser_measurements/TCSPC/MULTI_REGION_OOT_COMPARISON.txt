BLOCK 0 BASELINE: 17.73 cts/s (reference at 0 µW)

═══════════════════════════════════════════════════════════════════════════════

KEY FINDINGS:
─────────────

1. CONSISTENCY ACROSS REGIONS:
   All three OOT regions yield remarkably similar dark count estimates:
   - Mean range: 12.27-12.94 cts/s (variance: 0.67 cts/s, only 5.2%)
   - Std Dev range: 8.09-8.17 cts/s (variance: 0.08 cts/s, only 1%)
   - All show ~27-31% underestimation vs Block 0 baseline
   
   Interpretation: The histogram baseline is relatively uniform across the
   time range, suggesting no strong time-dependent artifacts in the 0-150 ns
   window.

2. REGION ORDERING BY ESTIMATE:
   Region 3 (<late>) < Region 1 (<early>) ≈ Region 2 (<middle>)
   12.27 < 12.94 ≈ 12.88 cts/s
   
   The >100 ns region gives 0.67 cts/s LOWER estimate than 0-10 ns.
   This small but systematic difference suggests slight time-dependent behavior.

3. SYSTEMATIC BIAS CHARACTERISTICS:
   - All regions show NEGATIVE bias vs Block 0 (underestimation)
   - Bias is NOT power-dependent (similar across all 43 measurements)
   - Bias is NOT time-window dependent (all regions show same pattern)
   - This supports a hardware-level baseline offset in the TCSPC acquisition

4. ROBUSTNESS OF OOT APPROACH:
   The consistency suggests out-of-time estimation is robust, BUT:
   - 27-31% systematic underestimation is significant
   - Block 0 baseline is single physical measurement (more trustworthy)
   - OOT method shows per-measurement variance (useful for uncertainty quantification)

═══════════════════════════════════════════════════════════════════════════════

PHYSICAL INTERPRETATION:
───────────────────────

The ~27% underestimation could be due to:
1. TCSPC baseline shift: Detector baseline not at exactly zero counts
2. Dead time effects: Some events lost in the pre-signal region
3. Pile-up in OOT region: Low-amplitude pulses in out-of-time not counted
4. Hardware offset: PicoHarp 300 acquisition with bias in histogram counting

The >100 ns region being slightly lower (-30.8% vs -27.0% for 0-10 ns)
suggests potential time-dependent dead-time recovery or baseline drift.
